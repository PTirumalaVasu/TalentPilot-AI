Story 4.0: YouTube IFrame Adapter & Watch Progress Capture

Overview

This demo showcases the PlayerAdapter abstraction pattern and CaptureService for auto-capturing video watch progress.

✓ What This Does:
  • Loads a YouTube video via IFrame
  • Polls playback position every 5–10 seconds
  • Queues samples and posts to backend every 12s or on 3+ samples
  • Flushes last position on tab close via navigator.sendBeacon()
  • Supports resume from saved watch position

Technical Details

/* PlayerAdapter Interface */ interface PlayerAdapter { position(): number duration(): number on(event: 'play'|'pause'|'ended'|'timeupdate', handler: () => void): void sendBeacon(position: number, eventTime: string): Promise<void> } /* YouTube Implementation */ - Uses YouTube IFrame API's polling-based getCurrentTime() - Polls every 5–10 seconds during playback - Listens to onStateChange events (PLAYING, PAUSED, ENDED) - Flushes position on tab close via navigator.sendBeacon()
🔬 API Proxy: The frontend is configured to proxy /api requests to http://localhost:8000. Make sure the backend is running on port 8000.

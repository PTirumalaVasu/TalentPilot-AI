## What to Build

### JavaScript only (no new HTML elements)

```javascript
// Modal: close on click-outside (click on the dark overlay itself, not the panel)
document.getElementById('assignment-details-modal').addEventListener('click', (e) => {
  if (e.target.id === 'assignment-details-modal') {
    closeDrillDown();
  }
});

// Modal: close on Escape key
document.addEventListener('keydown', (e) => {
  if (e.key === 'Escape') {
    const modal = document.getElementById('assignment-details-modal');
    if (!modal.classList.contains('hidden')) {
      closeDrillDown();
    }
  }
});

// User menu: close on click-outside
document.addEventListener('click', (e) => {
  const userMenu = document.getElementById('skills-dashboard-user-menu');
  const dropdown = userMenu.querySelector('[data-dropdown]');
  if (!userMenu.contains(e.target) && !dropdown.classList.contains('hidden')) {
    dropdown.classList.add('hidden');
  }
});
```

No Tailwind classes needed — this section is pure interaction wiring.

## Implementation Steps

### Step 1: Add modal click-outside listener
Attach to the overlay div (`#assignment-details-modal`) itself — checking `e.target.id === 'assignment-details-modal'` ensures clicks on the panel content (which stopPropagation isn't even needed for, since we check the exact target) don't trigger a close.

### Step 2: Add Escape key listener
Global `keydown` listener, checks modal isn't already hidden before closing (avoids redundant class toggling when modal is already closed).

### Step 3: Add user menu click-outside listener
Global click listener checks whether the click happened outside the user menu container.

### Step 4: Verify dev-mode overlay
No code change — confirm Ctrl+E toggle and Shift+Click ID copy still work (already included since Section 1).

## Common Issues & Fixes

### Issue: Clicking inside the modal panel also closes it
**Symptom**: Any click anywhere closes the modal, including clicks on the explanation text
**Cause**: Click listener checks the wrong target (e.g., checking `contains` instead of exact `id` match, or listener attached to `document` instead of the overlay)
**Fix**: Ensure the check is `e.target.id === 'assignment-details-modal'` (the overlay itself is the only valid close target) — the panel div is a separate nested element, so clicks on it never equal the overlay's id.

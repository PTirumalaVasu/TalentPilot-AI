### JavaScript

```javascript
async function openDrillDown(assignmentId) {
  const a = await PrototypeAPI.getAssignmentDetail(assignmentId);

  document.getElementById('assignment-details-header-id').textContent =
    `Employee: ${a.employee.firstName} ${a.employee.lastName} | Skill: ${a.skill.name}`;

  document.getElementById('assignment-details-provenance-summary').innerHTML =
    provenanceBadge(a);

  document.getElementById('assignment-details-raw-signal').innerHTML = renderRawSignal(a);
  document.getElementById('assignment-details-explanation').textContent = renderExplanation(a);

  document.getElementById('assignment-details-modal').classList.remove('hidden');
}

function closeDrillDown() {
  document.getElementById('assignment-details-modal').classList.add('hidden');
}

function renderRawSignal(a) {
  const rows = [`<div><strong>Assigned:</strong> ${a.assignedDate}</div>`];
  if (a.provenance === 'Verified') {
    rows.push(`<div><strong>Watch activity:</strong> ${a.watchPercent}% watched, auto-verified</div>`);
    rows.push(`<div><strong>Last activity:</strong> ${a.lastUpdate}</div>`);
  } else {
    rows.push(`<div><strong>Watch activity:</strong> ${a.watchActivity || 'None recorded'}</div>`);
    if (a.lastSelfReport) {
      rows.push(`<div><strong>Last self-report:</strong> "${a.lastSelfReport.value}" (entered ${a.lastSelfReport.enteredOn.split('T')[0]})</div>`);
    }
    rows.push(`<div><strong>Status:</strong> ${a.lastUpdate}</div>`);
  }
  return rows.map(escapeHtmlRow => escapeHtmlRow).join('');
}

function renderExplanation(a) {
  if (a.provenance === 'Verified') {
    return "This skill's progress is auto-verified from video watch activity — no self-reporting involved.";
  }
  if (a.provenance === 'Needs Attention') {
    return `This skill was assigned on ${a.assignedDate}. The last self-reported update said "${a.lastSelfReport?.value || 'in progress'}," but that entry hasn't been updated since. The row is marked "needs attention" because the signal is stale and current progress can't be confirmed.`;
  }
  return `This skill was assigned on ${a.assignedDate}. No video watch activity has been recorded since assignment. The employee's last self-reported update said "${a.lastSelfReport?.value || 'in progress'}," but that entry hasn't been updated since. The row is marked stale because we can't confirm current progress.`;
}
```

## Changes from Original Plan

### Issue: Dashboard stuck on Loading skeleton forever in a fresh browser tab

**Problem**: When testing Section 4 in a newly opened browser window, only the Loading skeleton was visible — the grid never appeared and drill-down was unreachable.

**Root cause**: `shared/prototype-api.js` seeded demo data via `fetch('data/demo-data.json')`. Browsers block `fetch()`/XHR requests to local files under the `file://` protocol (no local server was running — this prototype is meant to be double-click-openable). The fetch silently rejected, and `loadDashboard()` had no error handling, so execution stopped right after `setGridState('loading')`. Sections 2/3 appeared to work because they reused the same browser tab across refreshes, so sessionStorage stayed populated from one earlier successful load — a fresh tab (as opened for Section 4) had empty sessionStorage and hit the broken fetch path for the first time.

**Solution**: Replaced the fetch-based JSON load with a plain `<script>` include. Demo data now lives in `data/demo-data.js`, which assigns to `window.DEMO_DATA` — a normal script parse, not a network request, so it works identically under `file://` or a real server.

**Code change**:
- Added `data/demo-data.js` (converted from the now-removed `data/demo-data.json`)
- `shared/prototype-api.js`'s `_load()` now reads `window.DEMO_DATA` directly instead of `await fetch(...)`
- `01.1-Skills-Dashboard.html` now includes `<script src="data/demo-data.js">` before `shared/prototype-api.js`

**Learned**: This prototype format's core promise is "double-click to open, no server needed" — `fetch()` of local files breaks that promise even though it works fine when a dev server happens to be running. Any future prototype section that loads external JSON must use a `<script>`-tag global-variable pattern instead of `fetch()`, and this should be corrected in the shared `prototype-api.js` template guidance for future scenarios (02/03) too.

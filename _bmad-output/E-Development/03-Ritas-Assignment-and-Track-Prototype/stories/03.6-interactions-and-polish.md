## Goal

Wire `[+ New Assignment]` to actually open the Skill Assignment Flow modal (replacing the Scenario 01 stub), and extend Escape/click-outside handling to work correctly now that there are two modals on the same page.

---

## What to Build

### JavaScript — replace `handleNewAssignment()`

```javascript
function handleNewAssignment() {
  openAssignmentFlow();
}
```

### JavaScript — click-outside for the new modal

```javascript
document.getElementById('skill-assignment-flow-modal').addEventListener('click', (e) => {
  if (e.target.id === 'skill-assignment-flow-modal') {
    closeAssignmentFlow();
  }
});
```

### JavaScript — update the existing Escape handler to check both modals

```javascript
document.addEventListener('keydown', (e) => {
  if (e.key === 'Escape') {
    const provenanceModal = document.getElementById('assignment-details-modal');
    const assignmentModal = document.getElementById('skill-assignment-flow-modal');
    if (!provenanceModal.classList.contains('hidden')) {
      closeDrillDown();
    } else if (!assignmentModal.classList.contains('hidden')) {
      closeAssignmentFlow();
    }
  }
});
```

This **replaces** the existing Escape listener (from Scenario 01, which only checked the Provenance modal) rather than adding a second listener — avoids both modals trying to close on the same keypress if they were ever both open (shouldn't normally happen, but this makes the check explicit and correct either way).

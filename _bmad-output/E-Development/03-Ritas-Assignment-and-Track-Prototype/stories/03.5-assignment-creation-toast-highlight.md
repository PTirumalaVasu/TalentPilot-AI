# Story 03.5: Assignment Creation + Toast/New Row Highlight

**Section**: 5 of 6
**Complexity**: Medium

## Goal

Wire `handleAssign()` to actually create the assignment record, close the modal, refresh the dashboard grid, show a confirmation toast, and highlight the new row — closing the assignment-to-tracking loop end-to-end.

---

## What to Build

### HTML Elements (toast, added near the end of `<body>`)

```html
<div id="assignment-confirmation-toast" data-object-id="assignment-confirmation-toast"
     class="hidden fixed bottom-6 left-1/2 -translate-x-1/2 bg-gray-900 text-white px-6 py-3 rounded-lg shadow-lg z-50">
</div>
```

### JavaScript

```javascript
async function handleAssign() {
  const newAssignment = await PrototypeAPI.createAssignment({
    employeeId: assignmentFlowState.employeeId,
    skillId: assignmentFlowState.skillId,
    contentId: assignmentFlowState.content ? assignmentFlowState.content.id : null,
  });
  console.log('✅ Assignment created:', newAssignment);

  closeAssignmentFlow();
  await loadDashboard(); // re-render grid with the new row (existing Scenario 01 function)
  showToast(`✓ Skill assigned to ${newAssignment.employee.firstName} — ${newAssignment.skill.name}`);
  highlightRow(newAssignment.id);
}

function showToast(message) {
  const toast = document.getElementById('assignment-confirmation-toast');
  toast.textContent = message;
  toast.classList.remove('hidden');
  setTimeout(() => toast.classList.add('hidden'), 4000);
}

function highlightRow(assignmentId) {
  const row = document.querySelector(`tr[data-assignment-id="${assignmentId}"]`);
  if (row) {
    row.setAttribute('data-object-id', 'assignment-confirmation-new-row-highlight');
    row.classList.add('bg-talentpilot-50');
    setTimeout(() => row.classList.remove('bg-talentpilot-50'), 6000);
  }
}
```

Note: `rowHtml()` (from Scenario 01, reused here) already sets `data-assignment-id` on each `<tr>`, so `highlightRow()` can find the new row right after `loadDashboard()` re-renders.

## Scenario Entry Context (Phase 3)

**User Situation:** Rita is at her desk on Wednesday morning when a project lead Slack-messages asking "who can we staff on the Q3 skills initiative?" She needs to answer this readiness question in minutes with confidence.

**Mental State:**
- Hope: Open the dashboard and instantly trust which rows are verified so she can answer without a second thought.
- Worry: The dashboard looks professional but hides stale data, causing her to make a wrong staffing call.

**Discovery Method:** Rita clicks the TalentPilot-AI bookmark in her browser and opens the Assignment Dashboard.

### Accessibility Requirements

- Provenance labels use **color + text**, never color-only (WCAG AA compliant)
- Grid is keyboard navigable (drill-down arrows accessible via keyboard)
- User menu dropdown is accessible via keyboard (Escape closes, Tab/Shift-Tab navigate)
- All buttons have descriptive labels (not icon-only)

---

### Real-Time Updates Architecture

**For POC:** Data loads completely in milliseconds. No polling or WebSocket required yet.

**For future phases:**
- Consider WebSocket for live updates when Casey's watch progress changes
- Show subtle "Updated" indicator on affected rows
- Debounce updates (batch within 2-3 second window)

---

## Content Rules

**Provenance Labeling Non-Negotiables:**
- ✅ Always pair status text with a visual treatment (color + icon, not color-only).
- ✅ Use simple labels: "In Progress" or "Completed"
- ✅ Never abbreviate. Write "In Progress," not "IP."

**Copy Tone:**
- Clear, matter-of-fact, no fluff.
- HR-facing copy is factual and confident, never playful.

---

## Design Constraints

- **Responsive:** Desktop-primary (no mobile version needed; Rita always checks from her desk).
- **Performance:** Dashboard must load in < 2 seconds (real-time feel expected).
- **Real-time updates:** If Casey finishes a video while Rita is viewing the dashboard, the row should update automatically (or show a subtle refresh indicator).

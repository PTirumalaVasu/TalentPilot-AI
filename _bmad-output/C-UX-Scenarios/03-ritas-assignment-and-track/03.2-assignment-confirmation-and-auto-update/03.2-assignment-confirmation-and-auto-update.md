### Auto-Update Architecture (Technical)

**Real-Time Updates:**
- When Casey starts watching the assigned skill, backend captures watch-position
- Each write from Casey's player triggers dashboard row update
- Rita's row automatically updates: `Assigned · Awaiting first watch` → `In Progress · [%] watched`
- Rita does NOT need to refresh; dashboard reflects changes in real-time

**No Manual Intervention:**
- Rita never clicks "refresh" or "sync"
- Dashboard auto-updates as Casey engages
- This proves the signal pipeline works end-to-end

## On-Page Interactions (Storyboard Items)

**Interaction 1: Return to Dashboard**
- Rita's assignment modal closes.
- Dashboard reloads or updates to show the new assignment row.
- **Key UX moment:** The new row appears immediately. Rita doesn't have to refresh or navigate elsewhere. The assignment is live.

**Interaction 2: Visual Confirmation**
- Rita scans the grid and immediately sees the new row: "Casey | Python Basics | Assigned · Awaiting first watch."
- The row is visually distinct (highlighted or marked as "Just Added").
- **Key UX moment:** Rita has visual confirmation that her assignment was successful. She can stop worrying about whether the form submission worked.

**Interaction 3: Understand the Status**
- Rita reads the provenance label: `Assigned · Awaiting first watch`.
- She understands: This skill has been assigned but Casey hasn't started watching yet (0% progress).
- **Key UX moment:** The label is honest and clear. Rita knows exactly what state the assignment is in and what to expect next.

**Interaction 4: Optional — Drill-Down on New Row**
- Rita may click the drill-down arrow on the new row to see details.
- Drill-down shows: Assignment date (today), Content linked (Python Basics for Beginners · Approved), No watch activity yet.
- Rita closes the drill-down.
- (This is optional; not required for scenario completion.)

## Content Rules

**New Assignment Display:**
- ✅ New row appears immediately (no refresh required).
- ✅ Status label is `Assigned · Awaiting first watch` (clear, honest state).
- ✅ Progress bar is empty (0% — accurate representation of no activity yet).
- ✅ Visual distinction (highlight, badge, or message) confirms creation.
- ✅ Row is positioned logically in the grid (alphabetically by employee, chronologically by assignment date, or by user preference).

**Copy Tone:**
- Factual and confirming.
- "Awaiting first watch" is neutral language (not "pending" or "incomplete").

## Integration Notes for Build

**Assignment Creation:**
- When Rita submits the Skill Assignment Flow form (03.1), the backend creates a new `assignment` record with status = "Assigned."
- The backend immediately queries and returns the updated assignment list.
- The dashboard grid is re-rendered to include the new row.

**Real-Time Updates (Future):**

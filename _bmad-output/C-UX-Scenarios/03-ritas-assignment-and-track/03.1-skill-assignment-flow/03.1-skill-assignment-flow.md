### Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| **Step 1: Employee selection** | Modal just opened (happy path) | Employee dropdown active; Step 2/3 disabled until selection made | Select employee, close, cancel |
| **Step 2: Skill selection** | Employee selected | Skill search/list active; Step 3 disabled until selection made | Select skill, close, cancel |
| **Step 3: Content review** | Skill selected, content auto-linked | Recommended content shown; [Assign] enabled | Assign, choose different content, close, cancel |
| **Loading** | Employee list, skill list, or content recommendation is being fetched | Dropdown/list shows a brief loading placeholder in place of options | Wait; Cancel still available |
| **Empty — No content match** | Skill selected but recommendation engine returns no approved content | Step 3 shows: "No approved content found yet for this skill. [Choose Different Content] or assign without content." | Choose different content, assign without content, cancel |
| **Error** | Employee list, skill list, or content lookup fails to load | Inline message under the affected field: "Couldn't load [employees/skills/content]. [Retry]" | Retry, cancel |

**Component States:**
- Buttons: Enabled/Disabled based on form completion

## On-Page Interactions (Storyboard Items)

**Interaction 1: Open Assignment Form**
- Rita clicks [+ New Assignment].
- Modal/form opens showing Step 1: Select Employee.
- **Key UX moment:** The form is simple and linear. Rita knows exactly what she's doing at each step.

**Interaction 2: Select Employee (Step 1)**
- Rita clicks the employee dropdown.
- Searchable list appears or auto-complete field activates.
- Rita types "Casey" or selects from the list.
- Rita clicks [Continue to Skill Selection].
- Form advances to Step 2.

**Interaction 3: Select Skill (Step 2)**
- Rita sees the skill search field.
- Rita types "Python" or browses recommended skills.
- Rita selects "Python Basics."
- Rita clicks [Review Content].
- Form advances to Step 3.

**Interaction 4: Review AI-Recommended Content (Step 3)**
- Rita sees the recommended content: "Python Basics for Beginners."
- The system has automatically matched "Python Basics" skill to the top-approved content for that skill.
- Rita reviews the thumbnail, duration, description, and approval badge.
- **Key UX moment:** Rita doesn't need to manually search for or link content. The system did it automatically. This reduces the assignment time to < 2 minutes.

**Interaction 5: Optional Content Browse (Optional Storyboard Item)**
- If Rita wants to choose different content, she clicks [Choose Different Content].
- A secondary search/browse panel appears (not primary flow, but available for edge cases).
- Rita can search YouTube or select from a curated list of alternatives.
- Rita either selects a different content item or clicks "Use Recommended" to go back to the auto-linked default.

**Interaction 6: Confirm Assignment (Exit Action)**
- Rita reviews the summary: Employee = Casey, Skill = Python Basics, Content = Python Basics for Beginners (Approved).
- Rita clicks [Assign].
- Assignment is created and saved to the backend.
- Modal closes, returning Rita to the Assignment Dashboard.
- Exit: Navigate to 03.2 Assignment Confirmation & Auto-Update.

## Content Rules

**Form Design:**
- ✅ Linear, step-by-step flow (no branching until optional "Choose Different Content").
- ✅ Clear labels for each field.
- ✅ Auto-complete and pre-populated recommendations to speed up form filling.
- ✅ Always show AI-recommended content automatically; never leave Rita searching.
- ✅ Always pair content with approval provenance ("✓ Approved").

**Copy Tone:**
- Factual and efficient (HR-facing).
- "Who should learn this?" frames the question as adding value to the employee.
- "We've found the best match" positions the AI recommendation as helpful, not intrusive.
- No compliance language or mandatory tone.

## Implementation Notes for Build

**Employee Dropdown:**
- Populate from SAILS employee directory / HR system.
- Support search/auto-complete by name.

**Skill Selection:**
- Pre-populate with curated skill taxonomy (e.g., "Python Basics," "Advanced SQL," "Data Visualization Fundamentals").
- Support search/auto-complete.
- Optionally allow custom skill entry (advanced HR users).

**AI Content Recommendation:**
- On skill selection, query the backend: "What is the best-approved content for this skill?"
- Return the top result (single recommendation, not multiple choices).
- Display thumbnail, title, source, duration, approval badge.
- Optional: "Choose Different Content" link triggers a secondary search view (out of scope for this step).

**Content Approval Provenance:**
- All content items shown must have an explicit approval status ("✓ Approved" or "Pending Approval").
- Never recommend unapproved content.

**Form Validation:**
- Employee: required.
- Skill: required.
- Content: auto-selected (not user-modifiable in primary flow).
- Save assignment only when all required fields are filled.

---

## Design Constraints

- **Responsive:** Desktop-primary.
- **Modal vs. Page:** Designer choice; modal (centered pop-up) keeps focus on the assignment task; separate page allows more breathing room. Both work.
- **Form Speed:** Complete assignment in < 2 minutes (3 simple steps).
- **Accessibility:** Clear labels, keyboard navigation, logical tab order.

## Challenges & Growth Areas

### 1. Database Transaction Semantics (Discovered Cross-Epic Gap)

**Issue:** Story 3.1 (Assignments creation) was missing database commit logic  
**Root Cause:** Transaction pattern wasn't finalized when Epic 3 was reviewed  
**Discovery:** Found during Story 4-4 code review (anti-spoofing validation needed Session-to-Assignment identity tie)  
**Resolution:** Updated `core/db.py::get_db()` to commit-on-success; retroactively fixed Story 3.1  
**Lesson:** Architectural patterns must be documented *as they emerge*, not retroactively

**Action:** Before Epic 5, document database transaction semantics for all developers

---

### 2. conftest.py Database Pool Corruption (Pre-Existing)

**Issue:** `Base.metadata.drop_all()` on teardown wipes entire shared test database, desyncing alembic_version  
**Symptoms:** Tests pass individually; fail when run concurrently  
**Workaround:** Stories 3.3, 4-2, etc. used private test engines (doesn't use shared pool)  
**Impact:** Low (worked around successfully); High if not fixed (Epic 5's live-DB tests will surface it)  
**Timeline:** Will resurface during Epic 5's dashboard queries (joins across multiple tables)

**Action:** Fix conftest.py before Epic 5 kickoff (3–4 hours, Charlie)

---
